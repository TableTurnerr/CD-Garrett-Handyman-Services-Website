This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
pnpm install         # install dependencies
pnpm dev             # local dev at http://localhost:3000
pnpm build           # static export to ./out (lint + type-check + SSG)
pnpm run serve-out   # serve the built /out directory locally
pnpm lint            # next lint
node scripts/build-images.mjs   # regenerate logo/favicons + project photos from /google-maps-images
```

There is no test suite. Verification is `pnpm build` (lint + type-check + static export of every route) plus a visual check with `serve-out`.

## What this is

A static-exported Next.js 15 (App Router) marketing site for **CD Garrett Handyman Services LLC**, a family-owned handyman in Midlothian, TX serving Ellis & Dallas counties. Brand: the **"Honey-Do Crew"** beehive identity with the slogan *"We Do What Your Honey Can't Do."* Palette is honey-amber (`--color-primary: #C77F1A`, `--color-gold: #F2B01E`) on warm charcoal (`--color-text: #1C1A17`).

This project was adapted from a restaurant template; the architecture (static export, data-driven content, SEO-first JSON-LD) is preserved.

## Data layer is the source of truth (`/src/data/`)

Edit data once and it propagates across pages, JSON-LD schema, sitemap and footer.

1. **`business.ts`** — `BUSINESS`: NAP, hours, owner, geo, areas served, rating/review count, socials, `quoteUrl`, features. Imported nearly everywhere.
2. **`services.ts`** — `SERVICE_CATEGORIES`: service catalog grouped by category. Drives `/services`, the homepage featured services, and the `serviceCatalogSchema` JSON-LD.
3. **`specialties.ts`** — `SPECIALTIES`: 11 SEO deep-dive service pages. Drives `/specialties` and `/specialties/[topic]`. `relatedServiceNames` references item names in `services.ts`.
4. **`neighborhoods.ts`** — `NEIGHBORHOODS`: Ellis/Dallas-county service-area configs for `/near/[city]`. `popularServices` references service names.
5. **`faqs.ts`** — `FAQS`. Homepage `FAQSection` shows `FAQS.slice(0, 6)` and injects FAQPage schema. Answers may contain markdown-style `[text](/url/)` links.
6. **`reviews.ts`** — `REVIEWS` (curated testimonials) + `PRESS_QUOTES` (trust signals).
7. **`schema.ts`** — JSON-LD factories: `organizationSchema`, `websiteSchema`, `businessSchema` (HomeAndConstructionBusiness/GeneralContractor), `serviceCatalogSchema`, `serviceSchema(input)`, `faqSchema`, `breadcrumbSchema`, `articleSchema`, `webPageSchema`, `localBusinessSchema`.
8. **`metadata.ts`** — `createMetadata()` factory. Every page uses it; never hand-roll metadata.
9. **`dishes.ts` / `matrix.ts`** — DORMANT service×city mesh infrastructure (empty `MATRIX_ALLOWLIST`, no route file). Leave inert until local landing pages are expanded.

## Routes

`output: "export"` + `trailingSlash: true`. All internal links include the trailing slash.

Static: `/`, `/services/`, `/specialties/`, `/honey-do/`, `/get-a-quote/`, `/handyman-services/`, `/our-story/`, `/near/`, `/satisfaction-guarantee/`.
Dynamic (pre-rendered via `generateStaticParams`): `/specialties/[topic]` (from `SPECIALTY_SLUGS`), `/near/[city]` (from `NEIGHBORHOOD_SLUGS`).

New static routes must be added to `STATIC_ROUTES` in `src/app/sitemap.ts`; dynamic routes are sourced from their data file.

## SEO

Every page: `createMetadata()` for title/description/canonical/OG; relevant JSON-LD via `<SchemaInjector>`; `<BreadcrumbNav>` paired with an injected `breadcrumbSchema()`.

## Styling

Tailwind CSS 4, custom tokens in `src/styles/globals.css` (`@theme` + `:root`). Light theme only. Reusable classes: `.btn-primary`, `.btn-secondary`, `.btn-gold`, `.card`, `.container-pad`, `.section-pad`, `.eyebrow`. `Inter` (body/headings) + `Fraunces` (serif accent) via `next/font`.

## Images

Real photos live under `/public/Images/` (`hero.webp`, `about.webp`, `services/*`, `specialties/*`, `gallery/project-*`). Source photos are in `/google-maps-images`; the logo + favicons + all webp derivatives are generated by `scripts/build-images.mjs` from `/public/Images/logo.svg` (the beehive). Render images via **`SmartImage`** (shimmer skeleton; not `next/image`, since the static export sets `images.unoptimized`).

## Contact / env

Public contact + links come from `NEXT_PUBLIC_*` in `.env.local` (overrides the defaults in `business.ts`). The review-submission modal posts to `NEXT_PUBLIC_REVIEW_API_*` and is gated by Cloudflare Turnstile (`NEXT_PUBLIC_TURNSTILE_SITE_KEY`; test key `1x00000000000000000000AA` always passes). The quote form is mailto-based (no backend).
